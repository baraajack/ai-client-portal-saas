## Branch Naming

```txt
feat/feature-name
fix/bug-name
chore/task-name
```

## Commit Messages

Use conventional commits:

```txt
feat: add clients table
fix: protect admin route
chore: update dependencies
```

## Before Opening PR

Run:

```bash
npm run lint
npm run test:run
npm run build
```

## Pull Requests

Every PR should include:

- Summary
- Screenshots if UI changed
- Testing notes
- Security considerations
